# Yumemono Archive — Website

Static front-end for the Yumemono X community archive. Renders 1,349 posts,
965 media items, and 255 members — all mirrored to durable storage.

Paired with the pipeline: https://github.com/hazy2go/x-community-archive

## Stack

- Static HTML/CSS/JS (no framework) served from this repo's root
- Vercel Edge Function at `/api/v/[id].js` that proxies Google Drive video
  downloads so `<video>` can embed them cross-origin (strips Drive's
  `Cross-Origin-Resource-Policy: same-site` and forwards Range headers)
- Data baked into `data/archive.json` (all posts + authors + media refs)
  generated by `scripts/export_static.py` from the pipeline's SQLite

## Local development

```bash
# 1. Generate site/data/*.json from the pipeline SQLite
make export

# 2. Dev server with video proxy
make serve
open http://localhost:8787
```

Requires `../yume-archive-site/data/important-community.sqlite3` and
`drive-map.json` / `tco-map.json` from the pipeline side.

## Deploy

```bash
make deploy     # vercel --prod
```

The first-time Vercel link is `vercel` (interactive). Afterwards
`vercel --prod` pushes the static site + edge function.

## Media pipeline

Media lives in Drive; the exporter rewrites URLs to two Drive endpoints:

- Images → `https://lh3.googleusercontent.com/d/{FILE_ID}=w1200`
- Videos → `/v/{FILE_ID}` (proxied through this Vercel Edge Function)
- Video posters → `https://drive.google.com/thumbnail?id={FILE_ID}&sz=w1200`

964 / 965 media items currently Drive-hosted. One stale pipeline-side.

## Link expansion

`tco-map.json` resolves every `t.co` shortlink in captured tweet bodies to
its real destination URL (898 / 898 resolved). The exporter substitutes
in place at build time so the archive survives t.co going dark.
